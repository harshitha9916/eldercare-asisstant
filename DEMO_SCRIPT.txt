[0:20 — WHY AGENTS? — 30 seconds]
(Show cover_page_banner.png fullscreen)

NARRATION:
"You might ask — why not just build an app?

A traditional app forces a caregiver to navigate menus and fill forms. It can't understand 'my dad seems off today.' It can't route a medication question to the right specialist automatically. And critically — it can't know when to stop and ask for human approval.

Agents are different. They understand natural language, they coordinate intelligently across multiple domains, they act autonomously on safe tasks — and they pause on dangerous ones.

ElderCare Assistant is built on Google ADK 2.0. It uses four AI agents that work together as a team — but a human caregiver stays in control of every critical decision."

---

[0:50 — ARCHITECTURE — 55 seconds]
(Show architecture_diagram.png. Point to each section as you describe it.)

NARRATION:
"Here is how it's built.

Every message first hits a Security Checkpoint — implemented in pure Python, not an LLM. It scrubs personal information like phone numbers and Social Security numbers, detects prompt injection attacks, and routes medical emergencies directly to a 911 alert — in milliseconds, with no LLM call ever made.

Clean messages reach the Orchestrator Agent — the conversational core that understands the request and delegates to the right specialist.

Three specialist sub-agents handle the work: the Routine Agent for daily schedules and doctor visits, the Medication Agent for prescriptions, and the Wellness Agent for mood, pain, and sleep logging.

Each specialist talks to a local MCP server — five dedicated tools, and each agent only accesses the tools it needs. The medication agent cannot schedule doctor visits. The wellness agent cannot touch medications.

Before any critical change is committed, the workflow pauses at the Human-in-the-Loop approval node. The caregiver must explicitly say yes. There is no way for the AI to approve its own actions."

---

[1:45 — LIVE DEMO — 90 seconds]
(Screen recording of ADK playground at localhost:18081. Use large font, clean browser window.)

NARRATION:
"Let me show this working in real time.

First — adding a medication. I type:
'Please add a new medication: Vitamin D3, 2000 IU, once daily in the morning.'

The security checkpoint processes the text instantly. The orchestrator delegates to the Medication Agent, which calls the MCP update_medication_schedule tool.

And then — the system stops. It shows: 'Caregiver approval is required. Do you approve? yes or no?'

This is the Human-in-the-Loop interrupt. The entire workflow is paused at the OS level. No LLM can bypass this. I type 'yes' — and the medication is committed.

Now scheduling a doctor visit: 'Schedule a doctor visit with Dr. Adams next Monday at 2 PM.' Same flow. Routed to the Routine Agent, triggers the HITL pause, caregiver approves.

Now contrast that with a wellness log: 'Log that I slept 7 hours and had low pain today.' This runs instantly — no interruption. Fast and frictionless for safe, routine updates."

[3:15 — SECURITY DEMO — 35 seconds]
(Stay on playground, type the PII and emergency examples. Keep terminal visible so audit log JSON is visible.)

NARRATION:
"Now the security features in action.

I send: 'My father's number is 555-867-5309 — log that he slept 8 hours.'
Watch the terminal — it shows pii_scrubbed: true. The phone number never reached the LLM.

And if I type: 'My father is having chest pain!' — no LLM is called at all. The system immediately responds with the 911 alert. This is a hard-coded safety net that fires in milliseconds. In a real emergency, that speed matters."

[3:50 — THE BUILD — 35 seconds]
(Show GitHub repo briefly, then return to architecture diagram.)

NARRATION:
"The full system is built on Google ADK 2.0, FastMCP, FastAPI, and the Agents CLI.

The project was scaffolded end-to-end using Antigravity — Google DeepMind's agentic coding assistant — which helped design the workflow graph, implement the security checkpoint, debug a subtle MCP stdio stream issue on Windows, and write the documentation.

It's containerized with Docker and pre-configured for deployment to Google Cloud Run via the agents-cli deploy command. The entire codebase has 19 passing unit tests — covering every security rule — no API key needed to run them."
