[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at how the architecture actively enforces security and workflow execution.
Every user input passes through our Security Checkpoint first, where it immediately scrubs sensitive PII and blocks prompt injections.
For clean requests, the checkpoint routes control to our central Orchestrator Agent.
This orchestrator immediately analyzes the request and delegates tasks to our specialized sub-agents: the Routine Specialist, the Medication Coordinator, or the Wellness Companion.
Each specialist queries and updates data through our local Model Context Protocol server, which exposes dedicated database tools.
Crucially, before saving any critical changes, the graph routes execution through the Human-in-the-Loop approval node.
If the request alters medication or schedules appointments, the system halts and demands caregiver validation before proceeding.
Once verified, the final output node delivers the confirmation.

[2:00 — LIVE DEMO]
(switch to playground at localhost:18081)
Let's demonstrate how this works in real-time.
When I type: "Please add a new medication: Vitamin D3, 2000 IU, once daily in the morning."
The checkpoint processes the text, the orchestrator delegates it to the Medication Coordinator, and the coordinator calls the MCP update tool.
Because this changes a drug regimen, the system actively halts and prompts me: "A caregiver/family member approval is required to proceed with this request. Do you approve? (yes/no)".
Once I type "yes", the system commits the change to our secure database.
If I schedule an appointment, like: "Schedule a doctor visit with Dr. Adams next Monday at 2:00 PM", the Routine Specialist handles it and triggers the same caregiver verification flow.
However, if I type: "Log that I slept 7 hours and had low pain today", the Wellness Companion logs the metrics immediately without pausing, keeping standard status logging fast and frictionless.
