[0:50 — ARCHITECTURE — 60 seconds]
(Show architecture_diagram.png. Animate or highlight each section as you describe it.)

NARRATION:
"Let me show you how it's built.

Every message first hits a Security Checkpoint — implemented in pure Python, not an LLM. This means it is completely deterministic. It scrubs personal information like phone numbers and Social Security numbers, detects prompt injection attacks, and routes medical emergencies directly to a 911 alert — in milliseconds, with no LLM call.

Clean messages flow to the Orchestrator Agent — the conversational core that talks to the user and decides which specialist to call.

Three specialist sub-agents handle the work: the Routine Agent for daily schedules and doctor visits, the Medication Agent for prescriptions, and the Wellness Agent for mood, pain, and sleep logging.

Each specialist talks to a local MCP server — Model Context Protocol — which exposes five dedicated tools. Critically, each agent only gets access to the tools it needs. The medication agent cannot schedule doctor visits. The wellness agent cannot touch medications. This principle of least privilege is enforced at the toolset level.

Before any critical change is committed, the workflow pauses at the Human-in-the-Loop approval node. The caregiver must explicitly type 'yes' to proceed. There is no way for the AI to approve its own actions."

---

[1:55 — LIVE DEMO — 90 seconds]
(Screen recording of ADK playground at localhost:18081. Use large font, clean browser window.)

NARRATION:
"Let me show this working in real time.

First — adding a medication. I type: 'Please add a new medication: Vitamin D3, 2000 IU, once daily in the morning.'

Watch what happens. The security checkpoint processes the text instantly. The orchestrator recognizes this as a medication request and delegates to the Medication Agent, which calls the MCP update_medication_schedule tool.

And then — the system stops. It shows: 'A caregiver approval is required. Do you approve? yes or no?'

This is the Human-in-the-Loop interrupt. The entire workflow is paused at the OS level. No LLM can bypass this. I type 'yes' — and now the medication is committed.

Now let me show scheduling a doctor visit. 'Schedule a doctor visit with Dr. Adams next Monday at 2 PM.' — Same flow. Routed to the Routine Agent, calls add_doctor_visit, triggers the HITL pause, caregiver approves.

Now contrast that with a wellness log: 'Log that I slept 7 hours and had low pain today.' — This routes to the Wellness Agent, runs instantly, no interruption needed. Fast and frictionless for routine updates.
